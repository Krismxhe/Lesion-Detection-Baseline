# det-baseline

A fair multi-backbone object detection baseline built on **MMYolo + MMPreTrain + timm**.
Swap the backbone between ResNet-50, Swin-T, PVT-v2-B2, ConvNeXt-S, and EfficientNet-B3 while keeping the neck, head, loss, and training schedule identical.

## Architecture

```
Input (1024 × 1024)
        │
        ▼
┌────────────────────────────────────────────────────┐
│  Backbone  (swappable)                             │
│      ResNet-50 / Swin-T / PVT-v2-B2                │
│       ConvNeXt-S / EfficientNet-B3                 │
└──────────┬──────────┬──────────┬───────────────────┘
           │ C3       │ C4       │ C5
           ▼          ▼          ▼
┌────────────────────────────────────────────────────┐
│  Neck: CSPNeXtPAFPN                                │
│  in_channels=[C3, C4, C5]   out_channels=128       │
└──────────┬──────────┬──────────┬───────────────────┘
           │ P3       │ P4       │ P5
           ▼          ▼          ▼
┌────────────────────────────────────────────────────┐
│  Head: RTMDetSepBNHead  (fixed across all configs) │
│  feat=128   strides=[8, 16, 32]                    │
└────────────────────────────────────────────────────┘
```

| Config | Backbone | Pre-train | C3 / C4 / C5 channels |
|--------|----------|-----------|------------------------|
| `rtmdet_resnet50_dr.py` | ResNet-50 | torchvision ImageNet-1k | 512 / 1024 / 2048 |
| `rtmdet_swin_t_dr.py` | Swin-T | mmpretrain ImageNet-1k | 192 / 384 / 768 |
| `rtmdet_pvtv2_b2_dr.py` | PVT-v2-B2 | mmpretrain ImageNet-1k | 128 / 320 / 512 |
| `rtmdet_convnext_s_dr.py` | ConvNeXt-S | mmpretrain ImageNet-1k | 192 / 384 / 768 |
| `rtmdet_efficientnet_b3_dr.py` | EfficientNet-B3 | timm ImageNet-1k | 48 / 136 / 384 |

## 1. Dataset Preparation

Convert YOLO-format labels to COCO JSON (one-time step):

```bash
python tools/convert_yolo_to_coco.py --data-root dataset-example
```

This writes `dataset-example/annotations/{train,val,test}.json`.

**Using your own dataset:** update `data_root` in `configs/_base_/dr_detection.py`
and make sure the directory structure matches:

```
your-dataset/
├── annotations/   ← generated by convert_yolo_to_coco.py
├── images/
│   ├── train/
│   ├── val/
│   └── test/
└── labels/        ← original YOLO txt files
```

## 2. Training

```bash
# ResNet-50
python tools/train.py configs/rtmdet_resnet50_dr.py

# Swin-Transformer Tiny
python tools/train.py configs/rtmdet_swin_t_dr.py

# PVT-v2-B2  (ViT-like hierarchical transformer)
python tools/train.py configs/rtmdet_pvtv2_b2_dr.py

# ConvNeXt-Small
python tools/train.py configs/rtmdet_convnext_s_dr.py

# EfficientNet-B3  (via timm)
python tools/train.py configs/rtmdet_efficientnet_b3_dr.py
```

Common options:

```bash
# Custom work directory
python tools/train.py configs/rtmdet_resnet50_dr.py --work-dir work_dirs/run1

# Mixed precision — saves ~30% VRAM
python tools/train.py configs/rtmdet_resnet50_dr.py --amp

# Resume from the latest checkpoint
python tools/train.py configs/rtmdet_resnet50_dr.py --resume

# Override any config value without editing files
python tools/train.py configs/rtmdet_resnet50_dr.py \
    --cfg-options train_dataloader.batch_size=2 train_cfg.max_epochs=50

# Multi-GPU (4 GPUs, DDP)
torchrun --nproc_per_node=4 tools/train.py configs/rtmdet_resnet50_dr.py
```

Checkpoints and logs are saved to `work_dirs/<config_stem>/`.
The best checkpoint is saved as `best_coco_bbox_mAP_epoch_*.pth`.

## 3. Evaluation

```bash
# Evaluate on the validation split
python tools/eval.py configs/rtmdet_resnet50_dr.py \
    work_dirs/rtmdet_resnet50_dr/best_coco_bbox_mAP_epoch_100.pth

# Evaluate on the test split
python tools/eval.py configs/rtmdet_resnet50_dr.py \
    work_dirs/rtmdet_resnet50_dr/best_coco_bbox_mAP_epoch_100.pth \
    --split test

# Save per-image COCO predictions to a JSON file
python tools/eval.py configs/rtmdet_resnet50_dr.py checkpoint.pth \
    --out results/resnet50_val.bbox.json
```

Output includes COCO-style mAP / mAP50 / mAP75 and per-class AP.

## 5. Feature Map Visualization

Visualize raw backbone / neck feature maps to check whether the model
has learned meaningful semantic representations.

### Command-line

```bash
# 6-layer overview: backbone C3/C4/C5 + neck P3/P4/P5 (default aggregation: l2norm)
python tools/visualize_features.py \
    configs/rtmdet_resnet50_dr.py checkpoint.pth image.jpg

# Switch aggregation mode (l2norm | mean | max)
python tools/visualize_features.py ... --agg mean

# Channel grid — top-32 most-activated filters for neck P4
python tools/visualize_features.py ... --mode channels --layer neck_p4 --top-k 32

# Multi-backbone comparison (unified colour scale)
python tools/visualize_features.py --compare \
    --models resnet50:configs/rtmdet_resnet50_dr.py:ckpt1.pth \
             pvtv2:configs/rtmdet_pvtv2_b2_dr.py:ckpt2.pth \
    --compare-image image.jpg --layer neck_p4
```

Output PNGs are saved to `vis_output/` by default.

cd notebooks

## 7. FAQ

**`KeyError: 'YOLODetector'` or `'CSPNeXtPAFPN'`**
mmyolo is not registered. Make sure `import det_baseline` runs before any config is parsed (all tools do this automatically).

**`KeyError: 'mmpretrain.SwinTransformer'`**
Install mmpretrain: `mim install "mmpretrain>=1.2.0"`.

**`KeyError: 'mmdet.TIMMBackbone'`** (EfficientNet config)
Requires timm: `pip install timm>=0.9.0` and mmdet >= 3.3.0.

**`KeyError: 'mmpretrain.ConvNeXt'`** (ConvNeXt-S config)
Requires mmpretrain >= 1.2.0: `mim install "mmpretrain>=1.2.0"`.

**ConvNeXt-S produces flat / collapsed feature maps**
Check that `gap_before_final_norm=False` is set in the backbone config.
If it is `True`, global-average-pooling collapses the spatial dimensions before the neck receives them.

**Swin-T `window_size` error**
Ensure mmpretrain >= 1.2.0. If the error persists, add `pad_small_map=True` to the backbone config.

**Out of memory**
- Reduce batch size: `--cfg-options train_dataloader.batch_size=2`
- Enable AMP: `--amp`
- Swin-T gradient checkpointing: add `with_cp=True` to backbone dict

**Only 5 sample images — will training crash?**
Mosaic's `max_cached_images=20` repeats samples when the dataset is smaller, so training runs. Results have no statistical meaning on 5 images — replace with your full dataset and update `data_root`.

## 8. Adding a New Backbone

Register a custom backbone in `det_baseline/backbones/`, then reference it in a new config:

```python
# det_baseline/backbones/my_backbone.py
from mmyolo.registry import MODELS

@MODELS.register_module()
class MyBackbone(nn.Module):
    def forward(self, x):
        ...
        return c3, c4, c5   # tuple of 3 NCHW tensors
```

```python
# configs/rtmdet_my_backbone_dr.py
_backbone_out_channels = [64, 128, 256]
model = dict(
    backbone=dict(type='MyBackbone', ...),
    neck=dict(in_channels=_backbone_out_channels, out_channels=128, ...),
    ...
)
```
